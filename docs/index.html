Cross-Sectional Equity Signal Research

A study of whether classic price/volume signals predict the cross-section of US equity returns — evaluated the way a quant researcher would: point-in-time data, transaction costs, purged/embargoed cross-validation, Newey–West IC t-stats, factor-neutral alpha, and a multiple-testing-corrected deflated Sharpe ratio.

The headline isn't a Sharpe — it's judgment. On a current-snapshot universe one signal showed a +8.0%/yr factor alpha (t ≈ 3.6). Rebuilt on point-in-time index membership, that alpha collapses to −0.7%/yr (t ≈ −0.3) — it was survivorship bias, not skill. After costs, multiple testing, and survivorship control, no signal survives.

About this project

What I built, and what I learned from it.

I started this project to understand how quant researchers actually decide whether a trading signal is real, rather than whether it merely looks good in a backtest. I took six classic signals — 12–1 momentum, short-term reversal, low volatility, idiosyncratic volatility, 52-week-high proximity, and an illiquidity proxy — turned each into a dollar-neutral long–short portfolio, and tried to evaluate them as honestly as I could.

The most useful thing I learned came from a mistake I almost made. On a universe of today's S&P 500 members, the illiquidity signal looked genuinely good — a +8%/yr factor alpha with a t-stat around 3.6. It would have been easy to write that down as a finding. But that universe only contains companies that survived and stayed in the index, so I rebuilt the whole study on point-in-time index membership (the constituents as they actually were on each date, including the ones later dropped or delisted). The alpha disappeared. The "edge" was survivorship bias. That one before/after comparison taught me more about backtesting than any positive result would have.

The second lesson was statistical honesty: because I tried twelve configurations and picked the best, I learned to discount the Sharpe with a deflated Sharpe ratio that accounts for how many things I tried — and once I did, nothing was significant. So the final result is a clean null: after transaction costs, multiple-testing correction, and survivorship control, none of these simple signals earns a credible return. Being able to reach and report that conclusion is, I think, the actual skill.

Reproduce it

Every number above comes from a command on freely downloaded data — nothing is hardcoded.

# install
pip install -r requirements.txt

# headline study (point-in-time) and the old survivorship-biased run
python -m experiments.run_baseline --universe pit
python -m experiments.run_baseline --universe snapshot

# survivorship before/after, RL allocator, and an independent re-derivation
python -m experiments.run_comparison
python -m experiments.run_rl --universe pit --learner linucb
python -m experiments.verify_headline --universe pit

# rigor tests (look-ahead, costs, CV purge, IC sign, point-in-time membership)
pytest -q
